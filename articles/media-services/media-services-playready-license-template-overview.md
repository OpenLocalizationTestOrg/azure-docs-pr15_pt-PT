#<a name="media-services-playready-license-template-overview"></a>Descrição geral dos modelos licença PlayReady de serviços de multimédia

Azure dos serviços de multimédia agora fornece um serviço para fornecer o Microsoft PlayReady licenças. Quando tenta o leitor de utilizador final (por exemplo, o Silverlight) reproduzir o seu conteúdo PlayReady protegida, é enviado um pedido para o serviço de entrega de licenças para obter uma licença. Se o serviço de licenças aprovar o pedido, problemas de licença que é enviada para o cliente e pode ser utilizada para desencriptar e reproduzir o conteúdo especificado.

Serviços de multimédia também fornecem APIs que permitem-lhe configurar as licenças de PlayReady. Licenças contêm restrições que pretende para o runtime PlayReady DRM para impor a quando estiver a tentar para reproduzir protegido por conteúdo de um utilizador e os direitos.
Abaixo estão alguns exemplos de restrições de licença PlayReady que pode especificar:

- Data/hora a partir do qual a licença é válida.
- O valor de data/hora quando a licença expira. 
- Licença serão guardadas no armazenamento persistente no cliente. Licenças persistentes são normalmente utilizadas para permitir a reprodução de offline do conteúdo.
- O nível de segurança mínimas que tem de ter um leitor de para reproduzir o seu conteúdo. 
- O nível de proteção de saída para os controlos de saída para audio\video conteúdo. 
- Para mais informações, consulte a secção de controlos de saída (3,5) no documento [PlayReady conformidade regras](https://www.microsoft.com/playready/licensing/compliance/) .

>[AZURE.NOTE]Atualmente, só pode configurar PlayRight da licença PlayReady (este direito é necessário). O PlayRight dá-o cliente a capacidade a reprodução o conteúdo. O PlayRight também lhe permite configurar restrições específicas de reprodução. Para mais informações, consulte o artigo [PlayReadyPlayRight](media-services-playready-license-template-overview.md#PlayReadyPlayRight).

Para configurar licenças PlayReady através dos serviços de multimédia, tem de configurar o modelo de licença PlayReady de serviços de multimédia. O modelo é definido na XML.

O exemplo seguinte mostra o modelo mais simples (e mais comum) que configura uma licença de transmissão básica. Com esta licença, os seus clientes seria conseguir reproduzir conteúdo do seu PlayReady protegido.
    
    <?xml version="1.0" encoding="utf-8"?>
    <PlayReadyLicenseResponseTemplate xmlns:i="http://www.w3.org/2001/XMLSchema-instance" 
                                      xmlns="http://schemas.microsoft.com/Azure/MediaServices/KeyDelivery/PlayReadyTemplate/v1">
      <LicenseTemplates>
        <PlayReadyLicenseTemplate>
          <ContentKey i:type="ContentEncryptionKeyFromHeader" />
          <PlayRight />
        </PlayReadyLicenseTemplate>
      </LicenseTemplates>
    </PlayReadyLicenseResponseTemplate>

XML está em conformidade com o esquema XML do PlayReady licença modelo definido no modelo de licença de PlayReady secção de esquema XML.

Dos serviços de multimédia também define um conjunto de classes .NET que podem ser utilizadas para feitos em série e anular a serialização de XML e para. Para descrição de classes principais, consulte o artigo [classes .NET de serviços de multimédia](media-services-playready-license-template-overview.md#classes). que são utilizadas para configurar modelos de licença.

Para obter um exemplo de ponto a ponto .NET classes que utiliza para configurar o modelo de licença PlayReady, consulte o artigo [utilizar encriptação dinâmico da PlayReady e serviço de entrega de licenças](https://msdn.microsoft.com/library/azure/dn783467.aspx).

##<a id="classes"></a>Classes .NET de serviços de multimédia que são utilizadas para configurar modelos de licença

Seguem-se que as principais classes .NET são utilizadas para configurar modelos de licença PlayReady de serviços de multimédia. Estas classes mapear para os tipos de definidos no [PlayReady licença modelo XML schema](media-services-playready-license-template-overview.md#schema).

A classe de [MediaServicesLicenseTemplateSerializer](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.mediaserviceslicensetemplateserializer.aspx) é utilizada para serializar e serialização de e para o modelo de licença dos serviços de multimédia XML.

###<a name="playreadylicenseresponsetemplate"></a>PlayReadyLicenseResponseTemplate

[PlayReadyLicenseResponseTemplate](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.playreadylicenseresponsetemplate.aspx) - esta classe representa o modelo para a resposta enviada para o utilizador final. Contém um campo para uma cadeia de dados personalizados entre o servidor de licenças e a aplicação (poderá ser útil para lógica de aplicação personalizada), bem como uma lista de modelos de licença de um ou mais.

Esta é a classe de "nível superior" na hierarquia de modelo. Que significa que o modelo de resposta inclui uma lista de modelos de licença e os modelos de licença incluem (direta ou indiretamente) todas as classes que compõem os dados de modelo para ser serializado.


###<a name="playreadylicensetemplate"></a>PlayReadyLicenseTemplate

[PlayReadyLicenseTemplate](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.playreadylicensetemplate.aspx) - a classe representa um modelo de licença para a criação de licenças de PlayReady para ser devolvido aos utilizadores finais. Que contém os dados na chave de conteúdo na licença e qualquer direitos ou restrições a impostas pelo runtime PlayReady DRM ao utilizar a chave de conteúdo.


###<a id="PlayReadyPlayRight"></a>PlayReadyPlayRight

[PlayReadyPlayRight](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.playreadyplayright.aspx) - esta classe representa PlayRight de uma licença de PlayReady. Concede ao utilizador a capacidade de reprodução o conteúdo sujeito as restrições de zero ou mais configurado na licença do e no PlayRight (para reproduzir específicas de política). A política do PlayRight tem de fazer com restrições de saída que controlam os tipos de resultados de que o conteúdo pode ser reproduzido sobre e quaisquer restrições que devem ser colocadas no local ao utilizar uma saída dada. Por exemplo, se o DigitalVideoOnlyContentRestriction estiver ativada, em seguida, o runtime DRM só irá permitir o vídeo ser apresentado ao longo do saídas digitais (não permitidas analógicas saídas de vídeo para passar o conteúdo).

>[AZURE.IMPORTANT]Estes tipos de restrições podem ser muito poderosos, mas também podem afetar a experiência do consumidor. Se a protecção saída estiverem configuradas demasiado restritiva, o conteúdo poderá ser reproduzido no alguns clientes. Para mais informações, consulte o documento de [Regras de conformidade PlayReady](https://www.microsoft.com/playready/licensing/compliance/) .

Para um exemplo de níveis de proteção de que suporta o Silverlight, consulte o artigo: [Silverlight suporte para protecção de saída](http://go.microsoft.com/fwlink/?LinkId=617318).
