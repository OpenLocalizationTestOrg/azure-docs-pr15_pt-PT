# <a name="azure-cli-commands-in-resource-manager-mode"></a>Azure comandos de clip no modo de Gestor de recursos

Este artigo fornece opções para comandos de interface de comandos Azure (CLI) que normalmente, é utilizado para criar e gerir Azure recursos no modelo de implementação de Gestor de recursos do Azure e sintaxe. Aceder a estes comandos executando o clip no modo de Gestor de recursos (processador). Não se trata de uma referência completa e sua versão do clip poderá mostrar comandos ligeiramente diferentes ou parâmetros. Para obter uma descrição geral dos recursos Azure e grupos de recursos, consulte o artigo [Descrição geral do Gestor de recursos do Azure](../azure-resource-manager/resource-group-overview.md).  

Para começar a, primeiro, [instale o clip do Azure](../xplat-cli-install.md) e [ligar à sua subscrição do Azure](../xplat-cli-connect.md) com uma trabalho ou conta escolar ou uma identidade de conta Microsoft.

Sintaxe do comando atual e opções na linha de comandos no modo de Gestor de recursos, escreva `azure help` ou, para apresentar a ajuda para um comando específico, `azure help [command]`. Também encontre exemplos de clip na documentação para criar e gerir serviços Azure específicos.

Parâmetros opcionais são apresentados entre parênteses Retos (por exemplo, `[parameter]`). Todos os outros parâmetros são necessários.

Para além de parâmetros opcionais específicas do comando aqui documentados, existem três parâmetros opcionais que podem ser utilizados para apresentar resultados detalhados como opções do pedido e os códigos de estado. O `-v` parâmetro fornece registo verboso bem como a `-vv` parâmetro fornece ainda mais detalhadas registo verboso. O `--json` opção exporta o resultado no formato de json observou.

## <a name="setting-the-resource-manager-mode"></a>Definir o modo de Gestor de recursos

Utilize o seguinte comando para ativar os comandos do modo de Gestor de recursos do Azure clip.

    azure config mode arm

>[AZURE.NOTE] Modo de Gestor de recursos do Azure e no modo de gestão de serviço do Azure o clip são mutuamente exclusivos. Isto é, recursos criados no modo de um não podem ser geridos a partir de outro modo.

## <a name="azure-hdinsight-commands-to-manage-your-hdinsight-clusters"></a>Azure hdinsight: comandos para gerir os seus clusters HDInsight

    hdinsight config create [options] <configFilePath> <overwrite>
    hdinsight config add-config-values [options] <configFilePath>
    hdinsight config add-script-action [options] <configFilePath>

Exemplo: Crie um ficheiro de configuração que contém uma acção de script para ser executada quando criar um cluster.

    hdinsight config create "C:\myFiles\configFile.config"
    hdinsight config add-script-action --configFilePath "C:\myFiles\configFile.config" --nodeType HeadNode --uri <scriptActionURI> --name myScriptAction --parameters "-param value"

## <a name="azure-vm-commands-to-manage-your-azure-virtual-machines"></a>Azure vm: comandos para gerir as suas máquinas virtuais do Azure

**Criar uma VM**

    vm create [options] <resource-group> <name> <location> <os-type>

**Criar uma VM com recursos predefinido**

    vm quick-create [options] <resource-group> <name> <location> <os-type> <image-urn> <admin-username> <admin-password
    
>[AZURE.TIP]A partir do clip versão 0,10, pode fornecer um alias de breve como "UbuntuLTS" ou "Win2012R2Datacenter" como o `image-urn` para algumas imagens Marketplace populares. Executar `azure help vm quick-create` para as opções. Para além disso, começando com a versão 0,10, `azure vm quick-create` utiliza o armazenamento de premium por predefinição se estiver disponível na região selecionada.

**Listar as máquinas virtuais dentro de uma conta**

    vm list [options]

**Obter uma máquina de virtual dentro de um grupo de recursos**

    vm show [options] <resource-group> <name>

**Eliminar uma máquina de virtual dentro de um grupo de recursos**

    vm delete [options] <resource-group> <name>

**Encerramento uma máquina virtual dentro de um grupo de recursos**

    vm stop [options] <resource-group> <name>

**Reiniciar uma máquina de virtual dentro de um grupo de recursos**

    vm restart [options] <resource-group> <name>

**Iniciar uma máquina virtual dentro de um grupo de recursos**

    vm start [options] <resource-group> <name>

**Encerramento uma máquina virtual dentro de um grupo de recursos e lançamentos os recursos de cluster**

    vm deallocate [options] <resource-group> <name>

**Tamanhos de máquina virtual disponíveis de lista**

    vm sizes [options]

**Capturar VM como SO ou da imagem VM**

    vm capture [options] <resource-group> <name> <vhd-name-prefix>

**Definir o estado da VM para Generalized**

    vm generalize [options] <resource-group> <name>

**Veja a instância da VM**

    vm get-instance-view [options] <resource-group> <name>

**Permite-lhe para repor as definições de acesso de ambiente de trabalho remoto ou SSH numa máquina Virtual e para repor a palavra-passe da conta que tenha administrador ou sudo autoridade**

    vm reset-access [options] <resource-group> <name>

**Actualizar VM por novos dados**

    vm set [options] <resource-group> <name>
