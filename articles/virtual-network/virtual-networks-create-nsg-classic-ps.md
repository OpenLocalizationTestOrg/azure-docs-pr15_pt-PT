# <a name="how-to-create-nsgs-classic-in-powershell"></a>Como criar NSGs (clássico) no PowerShell

O exemplo PowerShell comandos abaixo esperar um ambiente simple já tiver criado com base no cenário acima. Se pretender executar os comandos, tal como estes são apresentados neste documento, primeiro a criar o ambiente de teste através da [criação de um VNet](virtual-networks-create-vnet-classic-netcfg-ps.md).

## <a name="how-to-create-the-nsg-for-the-front-end-subnet"></a>Como criar NSG para a sub-rede front-end
Para criar um NSG denominada com nome **NSG FrontEnd** com base no cenário acima, siga os passos abaixo:

1. Se nunca tiver utilizado o Azure PowerShell, consulte o artigo [como instalar e configurar o Azure PowerShell](../powershell-install-configure.md) e siga as instruções para o fim para iniciar sessão no Azure e selecione a sua subscrição.

3. Crie um grupo de segurança de rede com o nome **NSG FrontEnd**.

        New-AzureNetworkSecurityGroup -Name "NSG-FrontEnd" -Location uswest `
            -Label "Front end subnet NSG"

    Saída esperada:

        Name         Location   Label               
        ----         --------   -----               
        NSG-FrontEnd West US    Front end subnet NSG


4. Crie uma regra de segurança permissão do acesso a porta 3389 da Internet.

        Get-AzureNetworkSecurityGroup -Name "NSG-FrontEnd" `
        | Set-AzureNetworkSecurityRule -Name rdp-rule `
            -Action Allow -Protocol TCP -Type Inbound -Priority 100 `
            -SourceAddressPrefix Internet  -SourcePortRange '*' `
            -DestinationAddressPrefix '*' -DestinationPortRange '3389'

    Saída esperada:

        Name     : NSG-FrontEnd
        Location : Central US
        Label    : Front end subnet NSG
        Rules    :

4. Crie uma regra de segurança permissão do acesso a porta 80 da Internet.

        | Set-AzureNetworkSecurityRule -Name web-rule `
            -Action Allow -Protocol TCP -Type Inbound -Priority 200 `

            -DestinationAddressPrefix '*' -DestinationPortRange '80'

    Saída esperada:

## <a name="how-to-create-the-nsg-for-the-back-end-subnet"></a>Como criar NSG para a sub-rede back-end
3. Crie um grupo de segurança de rede com o nome **NSG-back-end**.

        New-AzureNetworkSecurityGroup -Name "NSG-BackEnd" -Location uswest `
            -Label "Back end subnet NSG"

    Saída esperada:

        Name        Location   Label              
        ----        --------   -----              
        NSG-BackEnd West US    Back end subnet NSG


4. Crie uma regra de segurança permissão do acesso a partir da sub-rede front-end à porta 1433 (porta predefinida utilizada pelo SQL Server).

        Get-AzureNetworkSecurityGroup -Name "NSG-FrontEnd" `
        | Set-AzureNetworkSecurityRule -Name rdp-rule `
            -Action Allow -Protocol TCP -Type Inbound -Priority 100 `
            -SourceAddressPrefix 192.168.1.0/24  -SourcePortRange '*' `
            -DestinationAddressPrefix '*' -DestinationPortRange '1433'

    Saída esperada:

        Name     : NSG-BackEnd
        Location : Central US
        Label    : Back end subnet NSG
        Rules    :

                                                           Prefix          Range         Address Prefix   Port Range

                   ALLOW VNET INBOUND   65000     Allow    VIRTUAL_NETWORK *             VIRTUAL_NETWORK  *              *

                   ALLOW INTERNET       65001     Allow    *               *             INTERNET         *              *       
                   OUTBOUND                                                                                                      
                   DENY ALL OUTBOUND    65500     Deny     *               *             *                *              *      

4. Crie uma regra de segurança a bloquear o acesso a partir da sub-rede à Internet.

        Get-AzureNetworkSecurityGroup -Name "NSG-BackEnd" `
        | Set-AzureNetworkSecurityRule -Name block-internet `
            -Action Deny -Protocol '*' -Type Outbound -Priority 200 `
            -SourceAddressPrefix '*'  -SourcePortRange '*' `
            -DestinationAddressPrefix Internet -DestinationPortRange '*'
